写在前面

前一阶段linux基础知识暂告一段落，下面开始介绍linux系统管理的基本知识。本节主要涉及w、 vmstat、 top、 sar 、nload 五个命令。

w 命令

w命令用于显示目前登入系统的用户信息，可得知目前登入系统的用户有哪些人，以及他们正在执行的程序。

命令行格式：

常用选项含义
-f 　开启或关闭显示用户从何处登入系统
-h 　不显示各栏位的标题信息列
-l 　使用详细格式列表，此为预设值
-s 　使用简洁格式列表，不显示用户登入时间，终端机阶段作业和程序所耗费的CPU时间
-u 　忽略执行程序的名称，以及该程序耗费CPU时间的信息
-V 　显示版本信息

w 命令能查到哪些信息

示例：

上图中，

第一行从左到右依次为：时间、系统运行时间、登陆用户数、平均负载。最值得关注的是平均负载，由三个数字组成，分别代表1分钟、5分钟、15分钟的系统负载；数字越大，说明服务器的压力越大。

第二行显示的是登陆名、登陆地址等信息。

判断服务器压力的方法：

将均衡负载数与逻辑cpu做比较，如果长时间大于cpu个数，就说明系统压力大；反之，说明cpu资源有富余。

这里需要补充了解一下系统逻辑cpu的查询方法：

系统cpu信息保存在 /proc/cpuinfo,系统内核从0开始，逐一列出。

查询系统有几个cpu，可以使用 cat /proc/cpuinfo | grep -c 'processor'

注：grep -c 表示打印符合条件的行数。

通过上图可以看到：

  • 系统逻辑cpu有1个；

  • 一分钟平均负载是0.00；

所以，结论就是现在cpu很闲，没事干。

在实际使用当中，cpu负载的理想状态：平均负载接近但不高于逻辑cpu的值；这样cpu既不至于闲置，也至于太多处理不过来。

查询系统运行时间和负载情况，还可以使用 uptime 命令，跟w 命令第一行的效果一摸一样，如下：

这里涉及一个知识点：tty 和pts ：

在计算机发展早期，为了让尽可能多的用户利用系统资源，使用自己的IO设备连接和操作主机，产生了终端terminal的概念，即计算机设置了很多终端，每个使用者通过自己的终端来连接和使用主机资源。

进入PC时代后，电脑资源越来越便宜，每个人都可以轻松拥有一台电脑而不用跟别人分享，但是linux系统依然保留了早期的这种设定。只不过，这里的终端是由mingtty软件虚拟出来的。

  • tty：通常，Linux发行版提供7个虚拟屏幕，1~6号是命令行终端，第7个是X- Window，即图形化界面。

  • pts ：通过终端，还可以虚拟出无限多个终端，编号从pts/0开始，以此类推；

注：

  1. 关于tty 的配置文件,CentOS 6.X 开始 TTY 的配置由 /etc/inittab更改为 /etc/init/start-ttys.conf(注：此处有误，正确的配置文件查到再补充)，可以通过编辑这个文件来修改默认的终端数；

  2. centos7 下切换命令行模式和图形模式的方法：必须在root用户下操作

vmstat 命令

vmstat是Virtual Meomory Statistics（虚拟内存统计）的缩写，可对操作系统的虚拟内存、进程、CPU活动进行监控。可以对系统的整体情况进行统计，但不足之处是无法对某个进程进行深入分析。

物理内存和虚拟内存：

学习vmstat 命令，首先需要了解物理内存和虚拟内存的区别。

物理内存：系统硬件提供的内存大小，是真正的内存；

虚拟内存：利用磁盘空间虚拟出的一块逻辑内存，用作虚拟内存的磁盘空间被称为交换空间（Swap Space）。linux下设置虚拟内存的方法见这里

物理内存的速度快，但是空间有限。为了最大化利用物理内存，系统会不时的进行页面交换操作，以保持尽可能多的空闲物理内存，即使并没有什么事情需要内存，Linux也会交换出暂时不用的内存页面，从而避免等待交换所需的时间。

vmstat 命令行格式：

常用选项含义
-a显示活动内页；
-f显示启动后创建的进程总数；
-m显示slab信息；
-n头信息仅显示一次；
-s以表格方式显示事件计数器和内存状态；
-d报告磁盘状态；
-p显示指定的硬盘分区状态；
-S输出信息的单位；

参数：

  • 事件间隔：状态信息刷新的时间间隔；

  • 次数：显示报告的次数；

vmstat 能查到哪些信息

vmstat 默认输出

vmstat [n]：n代表数字，每隔n秒输出一次，手动按ctrl+c结束。

vmstat [m] [n] : m、 n 代表数字，意思是每个m秒输出一次，一共输出n次；

procs进程相关信息

  • r ：run，表示运行队列，某一时刻CPU只能处理一个任务，其他的任务只能排队，这个状态就是r记录的状态；如果这个数值长时间大于服务器CPU，就说明cpu资源不够用了。

  • b： block，表示等待资源的进程数,资源指的是I/O 、内存等。b状态反应的是进程任务将结果写入磁盘的过程，如果这个数字长期大于1，说明磁盘写入频繁，需要关注。

memory 内存相关信息

  • swpd： 虚拟内存已使用的大小，单位为KB；

  • free： 空闲的物理内存的大小；

  • buff： 表示即将写入的缓冲内存大小；

  • cache ：表示从磁盘中读取的缓存大小；

swap内存交换情况

  • si ：每秒从虚拟内存写入内存的数据量；（相对内存而言，是in）

  • so ：每秒虚拟内存写入磁盘的大小；（相对内存而言，是out）

注：这两个数据与swap 强有关，如果频繁变化，说明内存不够用。

io 磁盘使用情况

  • bi ：读磁盘的数据量；（相对内存而言，是in）

  • bo ：写磁盘的数据量；（相对内存而言，是out）

注：如果这两个数值过大，说明磁盘在频繁读写；由于磁盘读写速度相对于内存、CPU速度很慢，必然影响系统性能，反映在数据上，b也会增大。

system系统采集间隔内的中断数

  • in ：每秒CPU的中断次数，包括时间中断

  • cs ：每秒上下文切换次数，例如我们调用系统函数、线程切换，都要进程上下文切换。这个值要越小越好，数值偏大表示你的CPU大部分浪费在上下文切换，导致CPU干正经事的时间少了，没有得到充分利用。

CPU使用状态

  • us：用户CPU时间百分比；如果长时间大于50%，说明系统资源不够。

  • sy ：系统CPU时间百分比，如果太高，表示系统调用时间长，比如IO操作频繁。

  • id ：空闲 CPU时间百分比；一般来说，id + us + sy = 100,一般我认为id是空闲CPU使用率，us是用户CPU使用率，sy是系统CPU使用率。

  • wa ：等待IO CPU时间；

  • st ：表示被偷走的CPU所占百分比，一般为0.

top 命令

top命令用于动态监控进程所占用的系统资源，每隔3秒钟更新一次，特点是会将占用CPU、内存磁盘I/O 最高的进程放在最前面。

命令行格式：

常用选项含义
-b以批处理模式操作；
-c显示完整的进程信息；
-d屏幕刷新间隔时间；
-I忽略失效过程；
-s保密模式；
-S累积模式；
-i<时间>设置间隔时间；
-u<用户名>指定用户名；
-p<进程号>指定进程；
-n<次数>循环显示的次数

top 能查看哪些信息

第一列：跟w 命令第一行、uptime 命令查询结果显示一致；
第二列：当前的系统任务情况：

linux共有5种状态，如下：

状态码状态具体描述
D不可中断uninterruptible sleep (usually IO) 收到信号不唤醒和不可运行, 进程必须等待直到有中断发生
R运行runnable (on run queue) 正在运行或在运行队列中等待
S中断sleeping 休眠中, 受阻, 在等待某个条件的形成或接受到信号
T停止traced or stopped 进程收到SIGSTOP, SIGSTP, SIGTIN, SIGTOU信号后停止运行运行
Z僵死a defunct (“zombie”) process 进程已终止, 但进程描述符存在, 直到父进程调用wait4()系统调用后释放
第三列：cpu使用情况

这里重点关注一下us这一项，也就是用户使用cpu的情况；

第四列：内存使用情况；
第五列：虚拟内存使用情况；

sar命令

sar（System Activity Reporter系统活动情况报告）是Linux 上最为全面的系统性能分析工具之一，可以从多方面对系统的活动进行报告，包括：文件的读写情况、系统调用的使用情况、磁盘I/O、CPU效率、内存使用状况、进程活动及IPC有关的活动等。

Sar是后台进程sadc的前端显示工具，与其他系统监控工具相比，它可以打印从当天零点开始到当前时刻到系统状态信息。

sar 安装

centos 7 默认没有sar，需要单独安装；安装名为“sysstat”的包后，sadc就会自动从内核收集报告并保存到/var/log/sa/ 目录下；

启动sadc(system activity data collector)服务(sysstat)，这样报告就会被写入到日志文件“/var/log/sa/saDD”中并且已经存在的文档将会被归档,DD表示日期（该报告默认保存一个月）

命令行格式

  • t为采样间隔，n为采样次数，默认值是1；

  • -o file表示将命令结果以二进制格式存放在文件中，file 是文件名。

  • options 为命令行选项，sar命令常用选项如下：

常用选项含义
-A所有报告的总和
-u输出CPU使用情况的统计信息
-v输出inode、文件和其他内核表的统计信息
-d输出每一个块设备的活动信息
-r输出内存和交换空间的统计信息
-b显示I/O和传送速率的统计信息
-a文件读写情况
-c输出进程统计信息，每秒创建的进程数
-R输出内存页面的统计信息
-y终端设备活动情况
-w输出系统交换活动信息

-r选项生成内存的使用情况报告

kbcommit和%commit列的数据包括了内存和交换区使用量的总和。

查看网络流量 ：sar -n {keyword} or {ALL}

-n参数有6个不同的开关：

  • DEV显示网络接口信息；

  • EDEV显示关于网络错误的统计数据；

  • NFS统计活动的NFS客户端的信息；

  • NFSD统计NFS服务器的信息；

  • SOCK显示套接字信息；

  • ALL显示所有5个开关。

使用-f选项可以查看某一天的网络流量历史

那么，从这张表能看出哪些有用的信息呢？

列名称含义
IFACE设备名称
rxpck/s每秒接收的包的数量，单位是个数，数值上万，说明网络异常；
rxkB/s每秒接收的数据量，单位是KB，数值大于5000000，说明网络异常；
txpck/s每秒发送的包的数量，单位是个数；
txkB/s每秒发送的数据量，单位是KB；
-q选项：查看平均负载

-u 选项：查看CPU资源监控

sar -u -o test 10 3 // 每10秒采样一次，连续采样3次，观察CPU 的使用情况，并将采样结果以二进制形式存入当前目录下的文件test中

需要重点关注的指标：

  • %iowait：显示用于等待I/O操作占用 CPU 总时间的百分比；若 %iowait 的值过高，表示硬盘存在I/O瓶颈；

  • %idle：显示 CPU 空闲时间占用 CPU 总时间的百分比；若 %idle 的值高但系统响应慢时，有可能是 CPU 等待分配内存，此时应加大内存容量；若 %idle 的值持续低于1，则系统的 CPU 处理能力相对较低，表明系统中最需要解决的资源是 CPU 。

-v 选项：进程、I节点、文件和锁表状态监控

输出项含义
dentunusd目录高速缓存中未被使用的条目数量
file-nr文件句柄（file handle）的使用数量
inode-nr索引节点句柄（inode handle）的使用数量
pty-nr使用的pty数量
-o选项保存sar命令的输出结果
-f选项读sar日志文件
要判断系统瓶颈问题，有时需几个 sar 命令选项结合起来

  • 怀疑CPU存在瓶颈，可用 sar -u（CPU的利用率） 和 sar -q（运行队列和交换队列的平均长度）等来查看；

  • 怀疑内存存在瓶颈，可用sar -B（分页统计报告）、sar -r（物理内存使用情况） 和 sar -W （页面交换情况）等来查看；

  • 怀疑I/O存在瓶颈，可用 sar -b（缓存使用情况）、sar -u （CPU利用率）和 sar -d （磁盘使用情况）等来查看；

查看sar日志

安装sar之后，在/var/log/sa下还会生成一个sar打头的文件。sa开头的文件是二进位制文件，只能用sar命令行sar -f [filename]查看；而以sar开头的文件是文本文件，可以用cat命令查看。
