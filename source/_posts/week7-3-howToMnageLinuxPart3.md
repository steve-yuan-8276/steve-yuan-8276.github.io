#### 配置DNS 域名解析

##### 神马是DNS

DNS，Domain Name System或者Domain Name Service（域名系统或者域名服务）。在计算机网络当中，每个设备都要有自己的ip地址，以便于互相之间识别和联系沟通。但是ip地址不方便记忆，于是就有了域名系统。

DNS的作用就是自动把域名地址翻译为IP地址，帮助你找打访问对象的真实ip地址。

域名设置常见有：

* A记录 (Address) ：用来指定主机名（或域名）对应的IP地址记录，通俗来说A记录就是服务器的IP；

* CNAME记录：CNAME (Canonical Name)记录，通常称别名指向。你可以用一个域名指向另一个域名，比如，可以将`a.com`指向`abcd.com` ,这样在浏览器输入`a.com`，就可以跳转到`abcd.com`；

**注：**一般来讲，DNS至少要设置两个，一个主力、一个备用；这样只要有一个可用，就不会影响域名的正常使用。

##### 如何配置DNS

方法一：修改 `/etc/resolv.conf` 

```
[root@stevey network-scripts]# vim /etc/resolv.conf
# Generated by NetworkManager
nameserver 129.29.29.29    //注意设置域名的格式，按照这个格式进行修改
nameserver 8.8.8.8
```

**注：**此方法是临时性的，重启计算机，`/etc/resolv.conf` 会被网卡配置文件覆盖，前面做得更改会失效。

方法二：修改网卡配置文件，全局修改

```
[root@stevey network-scripts]# vim /etc/sysconfig/network-scripts/ifcfg-ens33
```

修改下图中红框内的部分，按照格式要求进行修改

![](https://farm1.staticflickr.com/968/41247964194_0a42da5599_o.png)

方法三：修改 `/etc/hosts `文件

hosts文件是linux系统中负责ip地址与域名快速解析的文件，包含了ip地址和主机名之间的映射。

不同的linux版本，文件也可能不同，比如Debian的对应文件是/`etc/hostname`；centos的对应文件是 `/etc/hosts`；

```
[root@stevey network-scripts]# cat /etc/hosts
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
```

如上所示，`/etc/hosts`包括两个部分，前面是ip地址，后面是域名。其设置规则如下：

* 一个ip地址可用对应多个域名；

* 每行只能有一个ip地址，即ip地址是唯一的；

* 如果出现同一个域名对应不同的IP地址，centos6以前面的ip为准；centos7以后面ip的为准。

修改host文件

```
vim /etc/hosts        //编辑hosts文件，逐行增加即可。
```

### linux 防火墙

#### SElinux

SElinux, 全称Security Enhanceed Linux，字面意思是强化安全的Linux系统。它是linux上一个强化安全的子系统， 默认安装在 Fedora 和 Red Hat Enterprise Linux 上，Linux内核2.6版本后集成在内核中。

传统的Linux文件权限与账号的关系是自主访问控制的，即DAC，意思是说根据文件的rwx以及账户权限来管理文件的访问。这样做的风险在于，如果root账户密码泄露，系统安全就无从谈起。

SElinux 是委托访问控制，即MAC。在这种模式下，可以针对特定的进程与特定的文件资源来进行权限的控制，即便是root用户，权限也不一定是777，而是根据当时进程的设置而决定的。

目前SELinux支持三种模式：

* `enforcing` 强制模式：代表SELinux正在运行中，并且已经正确开始限制domain/type.

* `permissive`:宽容模式，代表SELinux正在运行之中，不过仅会有警告信息，而不会限制domain/type的访问。

* `disabled`：关闭，SELinux没有实际运行。

##### 查看SElinux 相关信息

######  `getenforce`：当前SELinux的状态

```
[root@stevey network-scripts]# getenforce

```

###### `sestatus`：查看SELinux的策略

```
[root@stevey network-scripts]# sestatus
SELinux status:                 enabled       //是否启动 SELinux enabled 表示启动；disabled表示关闭
SELinuxfs mount:                /sys/fs/selinux    //SELinux 相关文件资料挂载点
SELinux root directory:         /etc/selinux    
Loaded policy name:             targeted       //当前策略

Mode from config file:          enforcing
Policy MLS status:              enabled
Policy deny_unknown status:     allowed

```

##### 切换SELinux模式 `setenforce [参数]`

* `0` ：permissing，宽容模式；

* `1` ：enforcing，强制模式；


##### 关闭SElinux

selinux 配置文件为 `/etc/selinux/config` ，编辑该文件并重启即可

```
vim /etc/selinux/config

将SELINUX=enforcing 改为disabled；保存退出，并重启系统即可。
```

### 防火墙管理工具

为了保障系统安全，我们需要利用防火墙，通过预先设置的规则来监控进出流量，并执行相应的处理。

在centos6.x之前的系统中，采用的`iptables`；而centos7开始，使用`firewalld`。严格说来，`iptables`与`firewalld`都不是真正的防火墙，而是用来定义防火墙策略的管理工具。

* `iptables`服务会把配置好的防火墙策略交由内核层面的`netfilter`网络过滤器来处理；

* `firewalld`服务则是把配置好的防火墙策略交由内核层面的nftables包过滤框架来处理。

这两者的区别在于：

* `firewalld`可以动态修改单条规则，动态管理规则集，允许更新规则而不破坏现有会话和连接；`iptables`在修改了规则后必须得全部刷新才可以生效；

* `firewalld`使用区域和服务而不是链式规则；

* `firewalld`默认是拒绝的，需要设置以后才能放行；`iptables`默认是允许的，需要拒绝的才去限制；

#### netfilter的 5链、5表

前面说过，`netfilter`是系统真正的安全框架，在系统内核里；而`iptables`只是一个命令行工具，利用这个工具来操作`netfilter`

##### netfilter的 5个链

防火墙的核心就是设定规则，在不同的节点，对数据匹配规则并执行相应的操作。

链（chains）是数据包传播的路径，每一条链其实就是众多规则中的一个检查清单，每一条链中可以有一条或数条规则。

![](https://farm1.staticflickr.com/972/41924662192_70a6231098_o.png)

![](https://farm1.staticflickr.com/881/42311826141_04f0662203_o.jpg)

规则链则依据数据包处理位置的不同，可以分为以下5种：

* `PREROUTING`：在进行路由选择前处理数据包；

* `INPUT`：处理流入的数据包；

* `OUTPUT`：处理流出的数据包；

* `FORWARD`：处理转发的数据包；

* `POSTROUTING`：在进行路由选择后处理数据包；

##### netfilter的 5个表

相同规则的集合，叫做“表”，netfilter有也有5个表：

![](https://farm1.staticflickr.com/949/41971355321_f35e30511a_o.png)

###### 表一：Filter（过滤数据包）

主要跟进入Linux本机的数据包有关，是默认的table。

- `INPUT` 链：过滤所有目标地址是本机的数据包（对进入本机数据包的过滤）

- `OUTPUT` 链：过滤所有本机产生的数据包（对源地址得数据包的过滤）

- `FORWARD` 链：过滤所有路过本机的数据包（源地址和目标地址都不是本机的数据包)。

###### 表二：NAT（网络地址转换）

如果第一个数据包允许经行NAT或Masquerade,那么其它数据包都会被做相同的动作，也就是其他数据包不会被一个一个地NAT（属于一个流的包只会经过这个表一次）任何时候都不要在这个表任何一条链经行过滤。

包括三个动作：

* `DNAT`：改变数据包的目的地址使包能重路由到某台机器，使公网能够访问局域网的服务器

* `SNAT`: 改变数据包的源地址，使局域网能访问公网

* `MASQUERADE`:和SNAT一样使局域网能访问公网，无固定IP,使用PPP.PPPoE等拨号上网接入Internet

Nat表包含3条链：

* `PREROUTING`链 :数据包到达防火墙时改变包的目的地址。

* `OUTPUT`链：改变本地产生数据包的目标地址。

* ` POSTROUTING`链:在数据包离开防火墙时改变数据包的源地址。


###### 表三：Mangle：修改数据包，改变包头中内容(TTL, TOS,  MARK )

这个 table 主要是与特殊的数据包的路由标志有关，用于给数据包做标记，然后根据标记去操作相应的包。

* `TOS` :设置改变数据包的服务类型，不要使用TOS设置发往Internet的包除非打算依赖TOS来路由，不能在表中配置任何过滤。SNAT、DNAT

* `TTL`:改变数据包的生存时间，可以让数据包只有一个特殊的TTL，欺骗ISP,有些ISP并不希望多台计算机共享一个连接访问Internet,ISP通过检查一台计算机所发出的数据包是否含有不同的TTL来判断。

* `Mask`:给数据包设置特殊的标记，通过标记配置带宽限制和基于请求分类。

Mangle表的5条链：

* `PREROUTING`链：数据包进入防火墙后，路由判断之前改变数据包。

* `POSTROUTING`链：数据包确定了目标地址后，即路由判断前改变数据包。

* `OUTPUT`链：在数据包被确定目的地址前改变数据包

* `INPUT`链：在数据包进入本机后，应用程序接受之前改变数据包。

* `FORWARD`链：第一次路由判断之后，最后一次路由判断前改变数据包。

###### 表四：RAW

关闭nat表上启动的连接追踪机制；默认系统的数据包会被追踪，这样会消耗一定的资源，因此可以通过RAW表来指定某些端口不被追踪。

* `NEW` : 该包想要开始一个连接（重新连接或将连接重定向）

* `RELATED`:该包是属于某个已经建立的连接所建立的新连接。例如：FTP的数据传输连接就是控制连接所 RELATED出来的连接。`--icmp-type  0 ( ping 应答) 就是--icmp-type 8` (ping 请求)所RELATED出来的

* `ESTABLISHED` :只要发送并接到应答，一个数据连接从NEW变为ESTABLISHED,而且该状态会继续匹配这个连接的后续数据包。

* `INVALID`:数据包不能被识别属于哪个连接或没有任何状态比如内存溢出，收到不知属于哪个连接的ICMP错误信息，一般应该DROP这个状态的任何数据。 

###### 表五：security

用于强制控制的网络规则，不常用。

#### iptables的基本语法

命令行格式：

```
iptables [-t 表] [-命令] [匹配] [操作]
```

- `-t 表` :用于指定命令应用于哪个iptables内置表。

- 命令选项: 用于指定iptables的执行方式，包括插入规则，删除规则和添加规则

命令 |说明
---|---
-P  --policy     |   <链名>  定义默认策略
-L  --list      |  <链名>  查看iptables规则列表
-A  --append    |    <链名>  在规则列表的最后增加1条规则
-I  --insert     |   <链名>  在指定的位置插入1条规则
-D  --delete      |  <链名>  从规则列表中删除1条规则
-R  --replace      | <链名>  替换规则列表中的某条规则
-F  --flush         |<链名>  删除表中所有规则
-Z  --zero          |<链名>  将表中数据包计数器和流量计数器归零
-X  --delete-chain  |<链名>  删除自定义链
-v  --verbose       |<链名>  与-L他命令一起使用显示更多更详细的信息

- 匹配规则:指定数据包与规则匹配所具有的特征，包括源地址，目的地址，传输协议和端口号等

匹配规则|说明
---|---
-i --in-interface    |指定数据包从哪个网络接口进入，
-o --out-interface  |指定数据包从哪个网络接口输出
-p ---proto          |指定数据包匹配的协议，如TCP、UDP和ICMP等
-s --source         |指定数据包匹配的源地址
   --sport           |指定数据包匹配的源端口号
   --dport           |指定数据包匹配的目的端口号
-m --match           |指定数据包规则所使用的过滤模块

iptables规则常用动作

动作|说明
---|---
REJECT    |拦阻该数据包，并返回数据包通知对方
DROP  |丢弃数据包不予处理
REDIRECT  |将封包重定向到另一个端口（PNAT），之后继续比对其它规则
MASQUERADE |改写封包来源IP为防火墙的IP，可以指定port 对应的范围，进行完此处理动作后，直接跳往下一个规则链（mangle:postrouting）
LOG   |将数据包相关信息纪录在 /var/log 中，之后继续比对其它规则
SNAT |改写封包来源 IP 为某特定 IP 或 IP 范围，之后直接跳往下一个规则链
DNAT |改写数据包目的地 IP 为某特定 IP 或 IP 范围，之后会直接跳往下一个规则链
MIRROR  |镜像数据包，也就是将来源 IP与目的地IP对调后，将数据包返回
QUEUE   |中断过滤程序，将封包放入队列，交给其它程序处理
RETURN  |结束在目前规则链中的过滤程序，返回主规则链继续过滤
MARK 将封包标上某个代号，以便提供作为后续过滤的条件判断依据，之后继续比对其它规则

##### iptables常见用法

###### 查看及清除规则

- 查看规则

```
iptables -t nat -nvl    

注：
   -t 指定表；如果不加-t，默认显示filter表相关信息；
   -n 表示不针对ip反解析主机名；
   -L 表示列出；
   -v 表示列出详细信息；
   -nvL 表示查看该表格的详细规则；
   
```

- 删除规则

```
iptables -F    //删除所有规则


iptables -Z    //包及流量计数器归零

```

###### 增加删除指定规则

命令行格式： iptables [选项] 

常见选项| 说明
---|---
-A|追加一条规则
-D|删除一条规则
-I| 大写I，表示插入一条规则，实际效果跟-A一样
-p|小写p，指定协议，比如tcp、udp、icmp
-P|大写P，表示预设策略
--dport|跟-p连用，指定目标端口
--sport|跟-p连用，指定源端口
-d|指定目标ip
-s|指定源ip
-j|指定动作，如`ACCEPT`（允许）、`DROP`（丢弃）、`REJECT`（拒绝）
-i|指定网卡

示例：

```
iptables -I INPUT -s 8.8.8.8 -j DROP  //插入一条规则，将所有来自8.8.8.8的数据包丢弃

iptables -D INPUT -s 8.8.8.8 -j DROP  //删除上一条规则，注意删除规则时必须和插入规则保持一致

iptables -A INPUT  -s 1.1.1.1 -p tcp --dport 80 -j DROP   //插入一条规则，将来自1.1.1.1的tcp协议，发送到本机80端口的数据包丢弃

iptables -A INPUT -s 192.168.1.0/24 -i ens33 ACCEPT  //增加一条规则，将来自192.168.1.0/24这个网段，且作用在ens33上的包放行
```

此外，还有一种删除规则的方法：

- 先查询规则序号

```
iptables -nvL --line-numbers
```

- 删除置顶的规则

```
iptables -D [链名] [规则序号]
```
