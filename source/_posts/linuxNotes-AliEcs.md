## 写在前面

前一阵子，在老师的建议下买的阿里云主机一直闲置，趁着现在复习课程的机会，正好在云主机上实践一下。

## 云主机远程连接

阿里云服务器ECS连接提供了两种方式：

### 浏览器连接：

这种方式下，关键是要记住两个密码，一个是远程连接密码；另一个是云主机密码。这两者在管理台右侧选择`更多`就可以进行设置，之后重启云主机生效。

其他的就没什么了，需要连接的时候登陆控制台，点击远程连接，按提示操作就可以了，跟操作一个电脑没什么区别。

![](https://farm1.staticflickr.com/981/42146362951_fc1bf1e564_o.png)

<!--more-->

### 终端连接：

这个才是重点！

#### 账号密码登陆 

```
ssh root@[ECS公网ip]   //ECS公网ip在控制台进行查看
```

#### 秘钥登陆（推荐，设置好后可以免密登陆）

##### 第一步：生成秘钥

- 在控制台左侧点击 `秘钥对` --> `创建SSH秘钥对`

![](https://farm1.staticflickr.com/831/27275586607_c7c88c70d2_o.png)

- 按提示输入秘钥对名称，点击自动创建秘钥对，之后会自动下载一个`.pem`结尾的文件，**这个文件非常重要，妥善保管！**

![](https://farm1.staticflickr.com/964/28274009138_3afbdf1231_o.png)

- 回到之前秘钥对的那个界面，点击 `绑定秘钥对`，选择需要绑定的ECS云主机。

##### 第二步：采用秘钥登陆

```
ssh -i [.pem私钥文件] root@[公网IP地址]
```

**注：**

这里提到的`[.pem私钥文件]`，就是之前自动下载的那个文件，这里要写的是文件的绝对路径。那笔者来说，下载后保存在了`～/.ssh` 目录下，所以这里的路径就是 `~/.ssh/[filename]`

在这里还遇到了一个有意思的现象，第一次用秘钥登陆，出现错误提示：

![](https://farm1.staticflickr.com/962/28273649748_6d2f0976a8_o.png)

意思是说秘钥权限太open，重新修改为600权限后，顺利登陆。

```
chmod 600 ~/.ssh/ali-linux.pem
```

##### 第三步：设置alias

其实第二步结束就已经可以顺利登陆了，但是命令行太长了，还不完美。想到课程中曾经学过alias的用法，所有就有了第三步。

mac环境变量配置文件在`～/.bash_profile`,编辑这个文件，按照下面的格式设置alias，之后重新加载配置文件。

```
alias ali-linux='ssh -i [.pem私钥文件] root@[公网IP地址]'      //等号两边没有空格

```

之后，只要输入`ali-linux`，就可以登陆阿里云主机了，完美！
